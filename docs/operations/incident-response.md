# Incident Response

Last reviewed: 2026-03-23

## First actions

1. Stop the spread of the incident.
2. Revoke exposed credentials or sessions.
3. Preserve enough evidence to understand what happened.
4. Communicate status privately before any public write-up.

## JMCP-specific focus

- treat lost phones and compromised laptops as credential incidents
- assume logs, screenshots, and summaries may contain sensitive operational context
- prefer rotation and containment over quick reuse of compromised secrets
- record follow-up repo changes so the same class of incident becomes harder to repeat
