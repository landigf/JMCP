# Core Beliefs

Last reviewed: 2026-03-23

JMCP is being built for agentic development, so the repository itself must provide the map that a human teammate or coding agent needs.

## Working beliefs

- Repository-local knowledge beats chat history and private notes.
- `AGENTS.md` should stay short and point to deeper sources of truth.
- Plans are first-class artifacts for complex work.
- Mechanical enforcement is better than style guidance alone.
- Security defaults belong in code, docs, and CI from day one.
- Prefer boring, composable tools that future agents can reason about.
- Evidence matters. Changes should be validated with commands, tests, or observable outcomes.
